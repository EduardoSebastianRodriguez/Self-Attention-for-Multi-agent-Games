# Sec. IV-B — Multi-agent Nonlinear Navigation

This experiment reproduces **Fig. 3** (ours vs. DP-iLQR), **Fig. 4**
(qualitative trajectory comparison), **Fig. 5** (scalability GIFs to
`N ∈ {4, 8, 10, 15}`), and **Table I** (mean / std cost across 20 seeds)
of the paper. Seven unicycles, model-free policy gradient (Algorithm 1)
trained against an unknown nonlinear navigation cost.

The model-based baseline is **DP-iLQR** ([Williams & Deshmukh,
L-CSS 2024 / Williams, Chen, Mehr, ICRA 2023]; original code at
<https://github.com/labicon/dp-ilqr>, MIT). A minimal copy of the
`dpilqr` Python package is bundled under [`dpilqr/`](dpilqr/) so the
experiment is self-contained. The original DP-iLQR `LICENSE.txt` is
preserved at the root of this folder.

## Files

| File                          | Purpose                                                                                                                                                                |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `train_ours_navigation.py`    | Trains the self-attention policy with policy gradient (10 000 iters; new initial config every 100 iters). Internally runs DP-iLQR once for the baseline cost (`Js.npy`). Saves checkpoints / GIFs into `videos/`. |
| `run_dp_ilqr_baseline.py`     | Standalone DP-iLQR trajectory for one random 7-agent configuration. Useful for the top row of Fig. 4.                                                                  |
| `plot_training_curves.py`     | Reproduces **Fig. 3** (ours vs. DP-iLQR cost-vs-iteration). Loads from CWD if available, otherwise from `precomputed_losses/`.                                          |
| `evaluate_policy.py`          | Rolls out a trained checkpoint on a held-out random config and renders the trajectory + pairwise-distance plot — i.e. the bottom row of **Fig. 4**.                    |
| `scalability_gifs.py`         | Loads a 7-agent checkpoint and rolls it out at `N ∈ {4, 8, 10, 15}` to produce the **Fig. 5** GIFs.                                                                    |
| `scalability_table.py`        | 20-seed mean ± std of total cost normalized by area size, used to fill **Table I**.                                                                                    |
| `functions_batch.py`          | `SelfAttentionNonlinearPolicy`, `AttentionLayer`, `AttentionGain` and helpers used by all scripts.                                                                       |
| `dpilqr/`                     | Bundled DP-iLQR library (cost / dynamics / iLQR solver / plotting utilities).                                                                                          |
| `precomputed_losses/`         | `log_loss_pg_state.npy`, `Js.npy` from the run that generated Fig. 3.                                                                                                  |

## Reproduce

```bash
# 0. (one-time) build the cython extension that powers dpilqr's dynamics
python setup.py build_ext --inplace

# 1. Train OURS + DP-iLQR baseline (CPU, ~hours)
python train_ours_navigation.py
#    → log_loss_pg_state.npy, Js.npy in CWD
#    → checkpoints in videos/{n_agents}-unicycles_policy_{iter}.pth

# 2. Fig. 3 (training curves) — uses precomputed_losses/ until step 1 finishes
python plot_training_curves.py

# 3. Fig. 4 bottom row (trajectory of trained policy on a new initial config)
python evaluate_policy.py videos/7-unicycles_policy_9900.pth

# 4. Fig. 5 (scalability GIFs to 4/8/10/15 agents from the 7-agent checkpoint)
python scalability_gifs.py videos/7-unicycles_policy_9900.pth

# 5. Table I (normalized cost, mean ± std over 20 random seeds)
python scalability_table.py
```
